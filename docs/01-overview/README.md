# Overview & Intent

This section provides orientation and high-level understanding of the project.

## Purpose

When you're new to the project or need to understand the "why" behind decisions, start here. These documents explain:

- What the project is and what problem it solves
- High-level planning and implementation phases
- How to think about testing and verification

## Documents

- **[BUILD_PLAN.md](./BUILD_PLAN.md)** - Dependency-ordered implementation checklist with phases, verification steps, and testing requirements
- **[MANUAL_TESTING.md](./MANUAL_TESTING.md)** - Template and procedures for manual testing when automated tests aren't available

## When to Read

- **First time setup**: Read BUILD_PLAN.md to understand the project structure
- **Before making changes**: Check MANUAL_TESTING.md to understand testing requirements
- **Onboarding new contributors**: Start here for project orientation

## Reading Order

1. Start with BUILD_PLAN.md for the big picture
2. Reference MANUAL_TESTING.md when implementing features
