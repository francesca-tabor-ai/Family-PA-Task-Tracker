# Deployment

This section covers shipping the app to production and troubleshooting deployment issues.

## Purpose

When production is broken or you need to understand deployment issues, these documents provide answers. They document:

- Deployment procedures and workflows
- Common deployment issues and their fixes
- Diagnostic procedures for deployment problems
- Status tracking for deployment issues

## Documents

- **[VERCEL_DEPLOYMENT.md](./VERCEL_DEPLOYMENT.md)** - Complete guide for deploying the Next.js app to Vercel
- **[VERCEL_DEPLOYMENT_STATUS.md](./VERCEL_DEPLOYMENT_STATUS.md)** - Status tracking for deployment issues
- **[VERCEL_OUTPUT_DIRECTORY_FIX.md](./VERCEL_OUTPUT_DIRECTORY_FIX.md)** - Solution for "No Output Directory" errors
- **[VERCEL_404_DIAGNOSTIC.md](./VERCEL_404_DIAGNOSTIC.md)** - Diagnostic report for 404 errors during deployment

## When to Read

- **Deploying to production**: Follow VERCEL_DEPLOYMENT.md
- **Troubleshooting 404 errors**: Check VERCEL_404_DIAGNOSTIC.md
- **Output directory errors**: See VERCEL_OUTPUT_DIRECTORY_FIX.md
- **Understanding deployment history**: Review VERCEL_DEPLOYMENT_STATUS.md

## Reading Order

1. VERCEL_DEPLOYMENT.md for standard deployment
2. Diagnostic documents (VERCEL_404_DIAGNOSTIC.md, VERCEL_OUTPUT_DIRECTORY_FIX.md) when issues arise
3. VERCEL_DEPLOYMENT_STATUS.md for tracking resolution
